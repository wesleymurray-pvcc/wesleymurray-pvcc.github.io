Software Engineering The branch of computer science that seeks principles to guide the development of large, complex software systems.
Integrated Development Environment (IDE) Combine tools for developing software into a single, integrated package.
Association for Computing MAchinery (ACM) Founded in 1947 this is an international scientific and educational organization dedicated to advancing the arts, sciences, and applications of information technology.
Purpose of ACM World's largest educational and scientific computing society, uniting educators, researchers and professionals to inspire dialogue, share resources and address the field's challenges.

Purpose of IEEE It is the trusted voice for engineering, computing, and technology information around the globe.

Software Life Cycle: Development Phase The major steps in the traditional software development life cycle are requirements analysis, design, implementation, and testing.
Requirements Analysis Goal of which is to specify what services the proposed system will provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
Design Involves creating a plan for the construction of the proposed system.
Implementation Involves the actual writing of programs, creation of data files, and development of databases.
Testing Debugging programs and confirming that the final software product was compatible with the software requirements specification.
Modular Implementation The division of software into maneagable units, generically called modules, each of which deal with only a part of the software's overall responsibility.
Unified Modelling Language (UMI) Visual modeling language that helps software developers cisualize and construct new systems.
Use Case Interactions between a system and its users.
Use Case Diagram Depicts a proposed system in a large rectangeles with ovals and stick figures as representations of the use cases and actors.
Data Relationships: Single and ready to mingle.
One-To-One Exemplified by the association between between patients and occupied private rooms one room per patient.
One-To-Many One patient to one room and one physician to multiple patients in single rooms.
Many-To-Many Many patients and many physicians intermingling.
Design Pattern A predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) Groups charged with overseeing and enforcing he quality control system adopted by organizations.
Software Testing: Next to impossible to get 100% correct but an important part of the process still.
Glass Box Testing Software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
Black Box Testing Tests that do not rely on knowledge of the software's interior composition.
Beta Testing Introducing preliminary version to target audience for prerelease testing of the program.
Alpha Testing Same as beta but done on the developers system.
Documentation: Instruction Manuals for the software.
User Documentation Software instructions intended for the user.
System Documentation Describes internal composition of the software for future maintenance.
Technical Documentation Describes how software should be installed and serviced.
Intellectual Property Law Provides legal framework for proprietary ownership of software.
Software License Document where the rights of the owner are formally expressed in legal terms.
Software Disclaimer States limitations of software developers liabilities in regards to the software.
Best defenses against software liability claims (see last sentence in the chapter) To apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
